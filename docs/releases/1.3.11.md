# Release v1.3.11

## Resumen Ejecutivo
¡Bienvenido a la versión 1.3.11! En esta actualización, se implementaron mejoras significativas en la funcionalidad de descarga, optimizando la forma en que los archivos se manejan en el sistema. También se realizaron correcciones en el manejo de contextos y se mejoró el código para una mayor estabilidad. Ahora puedes disfrutar de un rendimiento más fluido y eficiente en tus aplicaciones.

## 🚀 Nuevas Funcionalidades

### Mejora en la Funcionalidad de Descarga
**Qué se implementó**: Se mejoró la funcionalidad de descarga con el uso de mocks en pruebas, facilitando la simulación de respuestas HTTP y mejorando la robustez de las pruebas.

**Para qué sirve**: Esto es especialmente útil para garantizar que tus descargas funcionen correctamente sin necesidad de depender de conexiones de red reales, lo que hace que las pruebas sean más rápidas y confiables.

**Cómo usarlo**: Aquí tienes un ejemplo de cómo puedes implementar esto en tus pruebas:

```typescript
import { EventEmitter } from 'events';
import proxyquire from 'proxyquire';
import sinon from 'sinon';
import { test } from 'uvu';
import * as assert from 'uvu/assert';

// Mocks de HTTP
const httpMock = {
    get: sinon.stub(),
};

// Importamos el módulo con mocks
const { generalDownload } = proxyquire('../../src/utils/download', {
    http: httpMock,
});

// Ejemplo de prueba
test('debe manejar descargas exitosas', async () => {
    httpMock.get.callsFake((url, callback) => {
        callback(null, { statusCode: 200, pipe: () => {} });
    });
    
    await generalDownload('http://example.com/file.txt');
    assert.ok(httpMock.get.calledOnce);
});
```

**Beneficios**: Ahora puedes ejecutar pruebas de descarga sin complicaciones, lo que acelera el desarrollo y mejora la cobertura de pruebas.

## 🐛 Correcciones de Bugs

### Corrección en el Manejo de Estados
**Qué problema se solucionó**: Se solucionó un problema donde el estado de flujo no se actualizaba correctamente al final de un proceso de mensaje.

**Qué causaba el problema**: El código anterior no gestionaba adecuadamente el final del flujo, lo que podría causar inconsistencias en el estado del bot.

**Cómo se corrigió**: Se eliminó la línea que intentaba actualizar el estado al final del flujo, lo que ahora evita que se dupliquen las actualizaciones innecesarias.

```diff
- await this.stateHandler.updateState({ from })({ __end_flow__: false });
```

**Beneficio para el usuario**: Ahora ya no ocurre el problema de estados inconsistentes, lo que significa que tu bot se comportará de manera más predecible y confiable.

## ⚡ Mejoras y Optimizaciones

### Optimización en la Clase Core
**Qué se mejoró**: Se simplificó el código dentro de `CoreClass` eliminando métodos innecesarios y mejorando la claridad del código.

**Por qué era necesario**: Anteriormente, había métodos que complicaban la gestión del estado y la construcción de contextos, lo que hacía que el código fuera más difícil de seguir y mantener.

**Beneficios prácticos**: Es más fácil entender y extender la funcionalidad de tu bot, lo que reduce el tiempo de desarrollo.

```typescript
// Antes
private buildCtxMethods = (): BotCtxMethods => ({
    endFlow: async (from: string) => {
        this.queuePrincipal.clearQueue(from);
    },
});

// Después
// Se eliminó el método innecesario para simplificar la clase
```
